**可以把“字段映射关系”抽到 JSON/YAML 配置里，但不建议把“全部解析逻辑”都交给配置。**
更推荐的是：**代码 + 配置混合模式。**

## 先回答你这个点

你现在这段：

```java
String sourceRecordId = requireText(row.get(0), "sourceRecordId", rowIndex);
String supplierName = requireText(row.get(1), "supplierName", rowIndex);
String sourceBusinessCode = requireText(row.get(2), "sourceBusinessCode", rowIndex);

```

本质上是“硬编码映射”：

- `row[0] -> sourceRecordId`
- `row[1] -> supplierName`
- `row[2] -> sourceBusinessCode`
- `row[3] -> supplierStatus`
- `row[4] -> sourceModifiedAt`

这当然可以改成配置驱动。

---

## 你说的两种方式，本质区别

### 1. 代码硬编码映射

就是你现在这种。

优点：

- 简单直接
- 性能好
- 调试方便
- 编译期更安全

缺点：

- 字段一变就要改代码
- 业务/实施人员不能自己调
- ERP 字段变化频繁时维护成本高

---

### 2. JSON/YAML 配置映射

比如写一个映射文件：

```json
{
  "erpType": "KINGDEE",
  "rowType": "array",
  "mappings": [
    { "target": "sourceRecordId", "sourceIndex": 0, "required": true },
    { "target": "supplierName", "sourceIndex": 1, "required": true },
    { "target": "sourceBusinessCode", "sourceIndex": 2, "required": true },
    { "target": "supplierStatus", "sourceIndex": 3, "required": true },
    { "target": "sourceModifiedAt", "sourceIndex": 4, "required": true }
  ]
}
```

解析器读取配置后动态做映射。

优点：

- 字段变更时不用改 Java 代码
- 可以让实施/配置人员参与维护
- 更适合多 ERP、多租户、多版本协议

缺点：

- 调试更复杂
- 配置写错了也会出问题
- 复杂转换不好表达
- 非技术人员其实也未必真能安全操作 JSON

---

## 关键点：哪些适合配置，哪些不适合配置

### 适合放配置里的

- 字段名对应关系
- 数组下标对应关系
- JSON 路径
- 是否必填
- 默认值
- 是否忽略某字段

比如：

- `row[2] -> sourceBusinessCode`
- `data.rows[*].code -> sourceBusinessCode`

---

### 不适合完全交给配置的

- 状态码转换
  例如 `A -> ACTIVE`, `B -> INACTIVE`
- 时间格式转换
  例如 `"2026-04-02 14:55:00"` 转 `Instant`
- 条件逻辑
  例如“如果税号为空但状态为 A，则补默认值”
- 数据清洗规则
- 异常处理策略
- 复杂嵌套结构展开

这些一般还是要靠代码。

---

## 所以最推荐的是：混合模式

### 代码负责

- 识别根结构
- 校验 payload 是否合法
- 遍历每一行
- 做复杂转换和业务规则
- 做异常处理

### 配置负责

- 字段映射关系
- 数组下标
- JSON path
- 必填/可选
- 默认值

这才是最稳的。

## 一个更现实的提醒

如果你真想让“非业务开发人员”操作：

**JSON 其实并不友好。**

对非开发人员更友好的通常是：

- YAML
- Excel
- 后台配置页面

因为 JSON 对他们来说仍然太技术化了，尤其是：

- sourceIndex
- jsonPath
- required
- transform

这些其实已经偏技术了。

所以更现实的说法是：

- `JSON/YAML` 适合“实施人员、配置人员、技术产品”
- 真正的纯业务人员，更适合后台表单配置界面
