服务端口是 `8924`，在 PowerShell 里必须用 `curl.exe`（避免命中 PowerShell 内置的 `Invoke-WebRequest` 别名）。

---

### 1. 查看 demo 信息

```powershell
curl.exe http://localhost:8924/api/examples/suppliers/demo
```

返回种子供应商列表、默认调度配置、示例请求路径。

---

### 2. 查看所有供应商连接状态

```powershell
curl.exe http://localhost:8924/api/examples/suppliers/connections
```

关注字段：`status` / `nextPullAt` / `leaseUntil` / `retryCount` / `version`

---

### 3. 查看单个供应商状态

```powershell
curl.exe http://localhost:8924/api/examples/suppliers/connections/9001
curl.exe http://localhost:8924/api/examples/suppliers/connections/9002
curl.exe http://localhost:8924/api/examples/suppliers/connections/9003
```

---

### 4. 手动触发一次调度（带请求体）

```powershell
curl.exe -X POST "http://localhost:8924/api/examples/suppliers/scheduling/dispatch" `
  -H "Content-Type: application/json" `
  -d "{\"batchSize\":10,\"leaseSeconds\":30}"
```

单行写法（复制更方便）：

```powershell
curl.exe -X POST "http://localhost:8924/api/examples/suppliers/scheduling/dispatch" -H "Content-Type: application/json" -d "{\"batchSize\":10,\"leaseSeconds\":30}"
```

---

### 5. 手动触发（不传请求体，走 yml 默认值）

```powershell
curl.exe -X POST "http://localhost:8924/api/examples/suppliers/scheduling/dispatch" -H "Content-Type: application/json"
```

---

### 推荐观察顺序

```powershell
# 1. 看初始状态
curl.exe http://localhost:8924/api/examples/suppliers/connections

# 2. 触发一次调度
curl.exe -X POST "http://localhost:8924/api/examples/suppliers/scheduling/dispatch" -H "Content-Type: application/json" -d "{\"batchSize\":10,\"leaseSeconds\":30}"

# 3. 再看状态变化
curl.exe http://localhost:8924/api/examples/suppliers/connections
```

触发后预期结果：

- `SUPPLIER_ALPHA (9001)` — `lastSuccessAt` 更新，`retryCount=0`，`nextPullAt` 往后推进
- `SUPPLIER_FAIL_ONCE (9002)` — 第一次失败，`lastErrorAt` 更新，`retryCount=1`，`nextPullAt` 按**指数退避**推进（约 30 秒后）
- `SUPPLIER_PAUSED (9003)` — 状态为 PAUSED，不被调度，字段不变
