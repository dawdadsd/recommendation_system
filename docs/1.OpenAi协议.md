### 思路接近了，但和 OpenAI 实际协议有差异。我带你看真实的 JSON 交互：

#### 真实协议：AI 要求调用工具时

AI 返回的 message 不是 `type: tool`，而是 `role: "assistant"` + `tool_calls` 字段：

```json
// 你发给 AI 的请求
{
  "messages": [
    { "role": "user", "content": "推荐一款手机" }
  ],
  "tools": [
    { "type": "function", "function": { "name": "searchItems", "parameters": {...} } }
  ]
}

// AI 返回的响应（注意：content 是 null，AI 没有直接回答）
{
  "choices": [{
    "message": {
      "role": "assistant",
      "content": null,
      "tool_calls": [
        {
          "id": "call_abc123",
          "type": "function",
          "function": {
            "name": "searchItems",
            "arguments": "{\"keyword\": \"手机\", \"maxPrice\": 5000}"
          }
        }
      ]
    }
  }]
}
```

#### 真实协议：你返回工具结果时

用 `role: "tool"` + `tool_call_id` 把结果喂回去：

```json
{
  "messages": [
    { "role": "user", "content": "推荐一款手机" },
    { "role": "assistant", "content": null, "tool_calls": [{"id": "call_abc123", ...}] },
    {
      "role": "tool",
      "tool_call_id": "call_abc123",
      "content": "[{\"name\": \"Pixel 9\", \"price\": 4999}]"
    }
  ]
}
```

#### 关键理解

整个过程有 4 种 role，不是 request/response 模型，而是**对话历史追加模型**：

`messages` 数组是一个不断增长的对话链：

- `[0] role: "system"` ← 你设定的人设
- `[1] role: "user"` ← 用户说的话
- `[2] role: "assistant"` ← AI 回复（可能含 `tool_calls`）
- `[3] role: "tool"` ← 你执行工具后的结果（通过 `tool_call_id` 关联）
- `[4] role: "assistant"` ← AI 拿到工具结果后的最终回答

**`tool_call_id` 是关键** —— 它把"AI 的请求"和"你的结果"一一对应起来。因为 AI 可能一次要求调用多个工具，每个都有独立的 id。
