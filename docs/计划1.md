# user_cf_recall 历史版本清理 + trainer 模块健康检查

## Context

ALS 训练每次运行都会向 `user_cf_recall` 表写入新版本的召回数据（model_version 格式如 `als_20260320120000`），但旧版本数据不会被删除，会导致表持续膨胀。同时 trainer 模块缺少健康检查端点，无法监控服务状态。

## 功能一：user_cf_recall 历史版本清理

### 设计思路

- `recommendation_model_version` 表记录了 `current_version` 和 `previous_version`
- 清理策略：保留当前版本 + 上一版本（安全回滚），删除其他所有旧版本
- 参考 backed 模块的 `PreferenceDeltaCleanUpService` 的批量删除模式（batch + rounds）
- 提供 REST API 手动触发（与训练接口并列）

### 新增/修改文件

**1. Domain 层 - 扩展 Repository 接口**

`UserCfRecallWriteRepository.java` — 新增方法：

```java
int deleteByModelVersionNotIn(List<String> keepVersions, int batchSize);
```

`RecommendationModelVersionRepository.java` — 新增方法：

```java
Optional<String> findPreviousVersion(String modelName);
```

**2. Infrastructure 层 - JDBC 实现**

`JdbcUserCfRecallWriteRepository.java` — 实现批量删除：

- SQL: `DELETE FROM user_cf_recall WHERE model_version NOT IN (?, ?) LIMIT ?`
- 利用已有的 `idx_model_version` 索引

`JdbcRecommendationModelVersionRepository.java` — 实现查询 previous_version

**3. Application 层 - 清理服务**

新建 `RecallCleanupService.java`（在 `application/service/` 下）：

- 查询当前版本 + 上一版本
- 批量循环删除不在保留列表中的行
- 配置项：`recommendation.recall-cleanup.batch-size` (默认 5000)
- 返回 `RecallCleanupResultDTO`

新建 `RecallCleanupResultDTO.java`（在 `application/dto/` 下）：

- 字段：modelName, keptVersions, batchSize, rounds, deletedRows, cleaned

**4. Interfaces 层 - REST 端点**

新建 `RecallCleanupController.java`（在 `interfaces/rest/` 下）：

- `POST /api/trainer/recall/cleanup` — 触发清理
- `GET /api/trainer/recall/versions` — 查看当前版本信息（可选，复用已有）

## 功能二：trainer 模块健康检查

### 设计思路

- 添加 `spring-boot-starter-actuator` 依赖
- 自定义 `HealthIndicator` 检查：数据库连通性 + ALS 训练状态
- 暴露 `/actuator/health` 端点

### 新增/修改文件

**1. pom.xml** — 添加 actuator 依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

**2. application.yml** — 配置 actuator + 清理参数

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info
  endpoint:
    health:
      show-details: always

recommendation:
  recall-cleanup:
    batch-size: 5000
```

**3. 自定义 HealthIndicator**

新建 `TrainerHealthIndicator.java`（在 `infrastructure/health/` 下）：

- 检查数据库连接（通过 JdbcTemplate 执行 `SELECT 1`）
- 报告 ALS 训练状态（IDLE / RUNNING / SUCCEEDED / FAILED）
- 报告当前活跃模型版本

## 验证方式

1. `mvn compile -pl recommendation-trainer -am` 编译通过
2. 启动应用后访问 `GET /actuator/health` 返回 UP + 训练状态详情
3. 调用 `POST /api/trainer/recall/cleanup` 能删除旧版本数据并返回清理结果
